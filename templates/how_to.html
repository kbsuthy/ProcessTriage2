How To Use This App

This page helps users understand how to think about processes that fit this tool.

Example Processes

Click any example to expand details.

Example 1: Creating a Promotional Flyer

Process Overview

Create a flyer to promote an event or initiative and distribute it through appropriate channels.

High-Level Steps

  • Identify the purpose of the flyer and the target audience
  • Gather required content (event details, messaging, branding assets)
  • Draft initial layout and visual concept
  • Review content for accuracy and brand alignment
  • Revise based on feedback
  • Finalize design
  • Distribute via print, email, web, or social channels

Typical Pain Points

  • Repetitive content edits
  • Manual approval cycles
  • Recreating similar designs repeatedly
Example 2: Publishing an Internal News Article

Process Overview

Publish an internal article to communicate updates, events, or announcements to employees.

High-Level Steps

  • Identify story topic and communication goal
  • Collect information and quotes from stakeholders
  • Draft article content
  • Edit for clarity, tone, and accuracy
  • Submit for review and approval
  • Revise based on feedback
  • Publish to internal platform
  • Share link through appropriate channels

Typical Pain Points

  • Multiple reviewers with overlapping feedback
  • Manual formatting and reposting
  • Tracking approval status
Example 3: Onboarding a New Employee

Process Overview

Prepare systems, access, and information needed for a new hire to begin work successfully.

High-Level Steps

  • Receive notification of new hire and start date
  • Create or request system accounts and permissions
  • Prepare equipment and workspace
  • Enroll employee in required systems and training
  • Send welcome and onboarding information
  • Conduct first-day orientation
  • Complete required documentation and acknowledgments
  • Monitor onboarding completion and follow up as needed

Typical Pain Points

  • Duplicate data entry across systems
  • Tracking completion across multiple tools
  • Timing dependencies between steps
Example 4: Processing a Service Request

Process Overview

Receive, evaluate, and respond to requests submitted by internal users.

High-Level Steps

  • User submits request through designated channel
  • Request is logged and categorized
  • Review request for completeness and eligibility
  • Determine priority and routing
  • Assign request to appropriate team or individual
  • Complete requested work
  • Communicate outcome to requester
  • Close request and record outcome

Typical Pain Points

  • Incomplete submissions
  • Manual routing decisions
  • Repetitive status updates
Example 5: Updating a Standard Operating Procedure (SOP)

Process Overview

Revise and publish updates to an existing process document.

High-Level Steps

  • Identify need for update or improvement
  • Review current documentation
  • Propose changes
  • Validate changes with subject-matter experts
  • Update documentation
  • Review for accuracy and consistency
  • Approve final version
  • Publish updated documentation
  • Notify affected stakeholders

Typical Pain Points

  • Version control issues
  • Repeated review cycles
  • Communicating changes effectively
Example 6: Launching a New Internal Tool or App

Process Overview

Plan, build, and roll out a new internal tool to support business needs.

High-Level Steps

  • Identify business need or problem
  • Gather requirements from stakeholders
  • Evaluate existing tools or solutions
  • Define scope and success criteria
  • Design solution
  • Build or configure tool
  • Test functionality and usability
  • Address defects or gaps
  • Prepare documentation and training
  • Launch tool
  • Monitor adoption and performance

Typical Pain Points

  • Scope creep
  • Manual testing and feedback collection
  • Coordinating across teams
Example 7: Building a New Software Application (End-to-End)

Process Overview

Design, develop, and deploy a new software system for organizational use.

High-Level Steps

  • Define problem statement and business objectives
  • Conduct stakeholder interviews and requirements gathering
  • Analyze feasibility and technical constraints
  • Design system architecture and user experience
  • Develop software components
  • Integrate with existing systems
  • Test functionality, security, and performance
  • Address defects and refinements
  • Deploy to production environment
  • Provide documentation and training
  • Monitor, support, and iterate

Typical Pain Points

  • Manual handoffs between teams
  • Repetitive testing activities
  • Inconsistent requirement documentation
Example 8: Approving a Purchase Request

Process Overview

Evaluate and approve requests to purchase goods or services.

High-Level Steps

  • Submit purchase request with justification
  • Review request for completeness
  • Validate budget availability
  • Route request for required approvals
  • Approve, deny, or request changes
  • Notify requester of decision
  • Initiate procurement if approved
  • Record transaction for reporting and audit purposes

Typical Pain Points

  • Approval delays
  • Re-entering information across systems
  • Lack of visibility into request status
